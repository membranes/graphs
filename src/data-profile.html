Annotation Schemes


						The development of token classification models requires the annotation of data entities.  An entity is a word, text piece.  Each entity belongs to a single class/category, e.g., river, lake, island, or other; in this context, river/lake/island are named entities, all other entities are other/miscellaneous/un-named entities.  Each named entity must be annotated according to a selected annotation scheme.  Annotation scheme examples:

						

							
								
								
							
							

								schemedescription
							

							
IO$I$: named entity tokens, $O$: other entities.

							
BIO$B$: denotes a token at the beginning of a named entity, $I$: denotes a token inside a named entity, $O$: represents tokens that are not part of a named entity

							
IOE$I$: denotes a token inside a named entity, $O$: denotes an un-named entity, $E$: denotes a token at the end of a named entity.

							
IOBES$I$: inside, $O$: outside/un-named, $B$: beginning, $E$: end, $S$: single-token entity

						


						


						This project uses the $BIO$ scheme.
