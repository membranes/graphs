Annotation Schemes & Modelling

The development of token classification models requires the annotation of data entities. An entity is a word, text piece. Each entity belongs to a single class/category, e.g., river, lake, island, or other; in this context, river/lake/island are named entities, all other entities are other/miscellaneous/un-named entities. Each named entity must be annotated according to a selected annotation scheme, in brief
  • The annotation tag of the first token of a named entity is $B$ (Begin).
  • If a named entity has more than one token, the last token may have annotation tag $E$ (End) or $I$ (Inside), depending on the schema; intermediate tokens will have annotation tag $I$.
  • If a token is not part of a named entity its annotation tag is $O$ (Outside).

Annotation scheme examples:
schemedescription
IOFor denoting $\rightarrow$ $I$: named entity tokens, $O$: other entities.
BIOFor denoting tokens $\rightarrow$ $B$: at the beginning of a named entity, $I$: inside
a named entity, $O$: that are not part of a named entity
IOEFor denoting tokens $\rightarrow$ $I$: inside a named entity, $O$: outwith a
named entity, $E$: at the end of a named entity
IOBES$I$: inside, $O$: outside, $B$: beginning, $E$: end, $S$: denotes a single-token entity

This project uses the $BIO$ scheme.
